I want a more succint, readable, and precise source, which Markdown is not. and I want typography features, which Markdown cannot express.

therefore, between these months, I keep wondering what an ideal proof presentation would be. I have the independent realization that traditional style of proof is in many ways inadequate. about the middle of August, I came across the Wikipedia post of Curry- Howard isomorphism, and that truly changed my worldview. I searched widely to learn more about logic, mathematical foundation, intuitionistic mathematics, constructivism, type theory, and interactive theorem provers. since this account is not centered about, so to say, these new mathematics, I shall continue my train of thought about the GitHub pages without explaining these terms.

in short, I had the urge to present classical mathematics in a more type theoretical style. a thorough approach might be designing a pseudo code for proof assistant, such as Coq, so that besides checking mentally, it is easy to rewrite in a way it can be checked via Coq too. additionally it is perhaps more asthetically pleasing to me to have a syntax which utilized only ASCII.

January, Febuary, and March soon passed, and it occurs to me classical mathematics has not missed much, if few little changes are made. just to be sure that I wasn't missing much, I took a look at several proposed mathematical foundation, such as the introduction and appendix of homotopy type theory, and the original papers of calculus of constructions. most importantly, I suggest that statements which are not identity nor partial orders, should not be buried in paragraphs, but occupy a lone line and numbered as equations. another suggestion is that when explaining the reason of a proposition, the reason should go before the equation, and these reasons are ordered according to the time of substitution. finally, application of law of excluded middle and nonconstructive axiom of choice should be announced every time. this way, it becomes likely to recover the Curry- Howard isomorphism.

therefore, I realized that classical mathematics is fine, as long as I have a strong inclination to type theoretical interpretation. a balance between constructiveness and convention is feasible. I may be very well content with more strictly natural deduction style presentation, to have a more reasonable scope of project.

it simply remains to design a markup which describes normal text and mathematics, and outputs HTML. this markup, like Markdown does, describes italic and bold texts, and so on. moreover, there has to be a mark which denotes the beginning and end of math modes, where I am free to rename the symbols as I wish. the code within the math mode, in turn, is to be convered to standard LaTeX, I am ready to code.
