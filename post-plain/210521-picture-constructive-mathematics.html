There are three kinds of object in the world: algorithms, problems, and machines. An algorithm can solve a problem, a problem can be computed on a machine, and a machine can simulate another machine.

Here \(\wp\) is an intuitionistic machine, which can only compute decidable algorithm. At any rate, we at least have \(\wp\) at our disposal. Put most stupidly, if we have \(\wp\), then we have \(\wp\). Namely:

\(\;\mathtt {Void}\;\)\(:\,\)\(\wp\)\(\:\Rightarrow\:\)\(\wp\)

Suppose machine \(\Gamma\) can simulate machine \(\Delta\) . If we have \(\Gamma\), and have any problem \(X\) which is computed on \(\Delta\), then we have a valid new computer stronger than \(\wp\). Namely:

\(\;\mathtt {Instantiation\_machine}\;\)\(:\,\)\(\;\mathtt {if}\;\)\(\Gamma\)\(\:\Rightarrow\:\)\(\Delta\)\(,\,\)

\(\;\mathtt {then}\;\)\(\;\mathtt {has}\;\)\(\varphi\)\(\left[ \Gamma ,\, X :\, D \right]\)\(:\,\)\(\wp\)\(;\,\)

Suppose computer \(\Gamma\) can compute problem \(X\) . If we have \(\Gamma\), and have any algorithm \(x\) which solves \(X\), then we have a valid new computer stronger than \(\wp\). Namely:

\(\;\mathtt {Instantiation\_problem}\;\)\(:\,\)\(\;\mathtt {if}\;\)\(\Gamma\)\(\:\Rightarrow\:\)\(X\)\(,\,\)

\(\;\mathtt {then}\;\)\(\;\mathtt {has}\;\)\(\varphi\)\(\left[ \Gamma ,\, x :\, X \right]\)\(:\,\)\(\wp\)\(;\,\)

Rules InstMachine and InstProblem correspond to universal instantiation in classical logic.

Suppose we have such a method, that, with machine \(G\), and with any .x an implementation of \(X\), we can simulate machine \(\Delta\) . Then, with \(G\), we obtain a method, by which any \(x\) an solution of \(X\) gives rise to a way to simulate \(\Delta\). Namely:

\(\;\mathtt {Deduction\_machine}\;\)\(:\,\)\(\;\mathtt {if}\;\)\(\psi\)\(\left[ \Gamma ,\, x :\, X \right]\)\(\:\Rightarrow\:\)\(\Delta\)\(,\,\)

\(\;\mathtt {then}\;\)\(\;\mathtt {has}\;\)\(\varphi\)\(\left[ \Gamma \right]\)\(\;=\;\)\(\vartheta\)\(,\,\)\(\vartheta\)\(\left[ x :\, X \right]\)\(\;=\;\)\(\Delta\)\(;\,\)

Suppose we have such a method, that, with machine \(G\), and with any .x an implementation of \(X\), we can solve problem \(Y\) . Then, with \(G\), we obtain a method, by which any \(x\) an solution of \(X\) gives rise to a way to solve \(Y\). Namely:

\(\;\mathtt {Deduction\_problem}\;\)\(:\,\)\(\;\mathtt {if}\;\)\(\psi\)\(\left[ \Gamma ,\, x :\, X \right]\)\(\:\Rightarrow\:\)\(Y\)\(:\,\)\(\wp\)\(,\,\)

\(\;\mathtt {then}\;\)\(\;\mathtt {has}\;\)\(\varphi\)\(\left[ \Gamma \right]\)\(\;=\;\)\(\vartheta\)\(,\,\)\(\vartheta\)\(\left[ x :\, X \right]\)\(:\,\)\(Y\)\(;\,\)

Rules DedMachine and DedProblem correspond to the deduction theorem in classical logic.
